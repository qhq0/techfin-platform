# 上传材料功能说明

> **对应需求**：业务需求说明书 — 企业材料上传与信息采集（步骤1）
>
> **参考 API 文档**：`api.json`（Postman Collection — dib_api）

---

## 一、核心业务逻辑详解

### 1.1 请求接收

#### 第一步：上传附件

```
POST /techfin/sxd/upload-attachment
Content-Type: multipart/form-data
```

| 表单字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `file` | File | 是 | 单个文件（PDF/Word/Excel/PPT/CSV/图片） |

**成功响应：** `{ "code": 0, "msg": "附件上传成功", "data": "1522260339629744128" }`

> 调用外部上传接口获取 `attId`，同时将文件元信息（attId、fileName、fileSize）写入 `application_att` 表。
> **不区分财务报表或商业计划书**，业务类型在提交资料时由 `financeFiles` / `businessFiles` 参数确定。

#### 第二步：删除附件

```
DELETE /techfin/sxd/delete-attachment/1522260339629744128
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `att_id` (路径变量) | String | 是 | 待删除的附件 ID，拼接到 URL 路径中 |

**成功响应：** `{ "code": 0, "msg": "附件删除成功", "data": null }`

> 从 `application_att` 表中删除对应记录，不影响已提交到外部系统的资料。

#### 第三步：提交资料

```
POST /techfin/sxd/submit-materials
Content-Type: application/json
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `creditCode` | String | 是 | 统一社会信用代码（18位数字或大写字母） |
| `customerNo` | String | 是 | 客户编号 |
| `financeFiles` | SubmitFileItem[] | 否 | 已上传的财务报表文件引用列表 |
| `businessFile` | SubmitFileItem | 否 | 已上传的商业计划书文件（仅允许一份） |

**SubmitFileItem 结构：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `attId` | String | 是 | 上传接口返回的附件 ID |
| `reportDate` | String | 否 | 报告日期（仅财务报表需要，来自前端输入） |

> `docName` 和 `docSize` 由服务层从 `application_att` 表根据 `attId` 查询，前端不再传入。
>
> **docName 唯一性：** 外部 API 要求 `docName` 全局唯一。服务层自动在原始文件名末尾附加 `attId` 后 6 位作为后缀（如 `财报.pdf` → `财报_a3f2c1.pdf`），无需前端处理。
>
> **docTypeId 获取方式：** `financeFiles` 中的文件 → 使用 `api.doc-type.finance` 配置值（`1273757631042949120`）；`businessFile` 中的文件 → 使用 `api.doc-type.business` 配置值（`1455619597903695872`）。

**请求示例：**
```json
{
  "creditCode": "91440101MA5XXXXXXX",
  "customerNo": "CUST2024001",
  "financeFiles": [
    { "attId": "1522260339629744128", "reportDate": "2024-12-31" }
  ],
  "businessFile": { "attId": "1522655358249328641" }
}
```

**成功响应：** `{ "code": 0, "msg": "资料提交成功", "data": "TASK-<32位hex>" }`

> `data` 字段直接返回任务 ID（`taskId`）字符串，前端可直接使用。
>
> **鉴权说明**：下游 API 所需的 `c1-token` 统一在 `application.properties` 中配置 `api.default-token`。
>
> **附件清理**：提交成功后，系统自动从 `application_att` 表中删除本次提交所引用的所有附件记录。

#### 第四步：查询资料提取状态

详见独立文档 [`docs/要素提取功能说明.md`](要素提取功能说明.md)，此处仅列出接口概要。**前置条件**：提交资料成功后获取 `taskId`。**推荐**：前端轮询该接口，直到 `completed=true` 后再进入后续流程。**后端注意**：`completed=true` 不代表所有文档提取成功，`F`（失败）也被视为结束状态。

### 1.2 处理流程

```
① 上传附件 → 调用外部附件上传 API 获取 attId → 写入 application_att（attId, fileName, fileSize）

② 提交资料 → 校验 creditCode/customerNo
         → 创建 application_record（taskId, status=UNFINISHED）
         → 遍历 financeFiles/businessFiles，分别获取对应 docTypeId
         → 从 application_att 查询每个 attId 对应的文件名/大小
         → 对 docName 附加 attId 后缀确保唯一性
         → 构建批量新增请求（docTypeId 根据 financeFiles/businessFiles 分类确定）
         → 调用外部资料批量新增 API → 解析响应中的 invalidDocNames 并告警
         → 创建 DocEntry（写入 application_doc，business_type=docTypeId）
         → 删除 application_att 中已提交的附件记录 → 返回成功

③ 查询提取状态 → 详见 [`docs/要素提取功能说明.md`](要素提取功能说明.md)
```

### 1.3 关键常量

| 常量 | 值 | 说明 |
|------|-----|------|
| `projectId` | `1521841328915996672` | 默认项目 ID |
| `dirId` | `0` | 默认目录 ID |
| `docTypeId` (财务报表) | `1273757631042949120` | 文档类型：财务报表 |
| `docTypeId` (商业计划书) | `1455619597903695872` | 文档类型：商业计划书 |
| `extraInfo` | `"{}"` | 扩展信息默认值 |

### 1.4 外部 API 调用

**附件上传** `POST /api/mdm/open/att/upload`

```
Request:  multipart/form-data, file=<文件流>
Headers:  c1-token: <从配置文件读取>
Response: { "success": true, "code": "1", "data": "1522260339629744128" }
```

**资料批量新增** `POST /api/extract/open/doc/batch/add`

```json
Request: [
  {
    "attId": "1522655358249328640",
    "dirId": 0,
    "docName": "商业计划书.pdf",
    "docSize": 791,
    "docTypeId": 1455619597903695872,
    "extraInfo": "{}",
    "projectId": 1521841328915996672,
    "reportDate": "2025-06-30"
  }
]
Headers: c1-token: <从配置文件读取>, Content-Type: application/json

Response: {
  "success": true,
  "code": "1",
  "data": {
    "docList": [{ "id": "1522656394616045568", "attId": "1522655358249328640", ... }]
  }
}
```

**资料详情** `GET /api/extract/open/doc/get/{doc_id}` — 详见 [`docs/要素提取功能说明.md`](要素提取功能说明.md#三外部-api-调用)

## 二、数据库表结构

### application_att 表（独立实体）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT (PK) | 自增主键 |
| `att_id` | VARCHAR(64) | 附件上传返回的附件 ID |
| `file_name` | VARCHAR(255) | 上传时的原始文件名 |
| `file_size` | BIGINT | 文件大小（字节） |
| `created_at` | DATETIME | 创建时间，用于定时清理未关联的孤立附件 |

> 该表为独立实体，上传时写入附件元信息（无业务类型字段），删除时从此表移除。提交资料时根据 `attId` 查询 `fileName` 和 `fileSize` 用于外部 API 调用。

### application_record 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | VARCHAR(64) (PK) | 任务 ID，提交时生成，格式 `TASK-<32位hex>` |
| `credit_code` | VARCHAR(18) | 统一社会信用代码 |
| `customer_no` | VARCHAR(64) | 客户编号 |
| `status` | VARCHAR(32) | 任务状态枚举：UNFINISHED（未完成）/ COMPLETED（已完成） |
| `act_cntlr_nm` | VARCHAR(200) | 实际控制人姓名，用户确认后回填 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

> 仅在用户点击"提交资料"时创建，主键为 `task_id`。无自增 id，无外键约束。

### application_doc 表（集合表，嵌入于 application_record）

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | VARCHAR(64) | 关联 `application_record.task_id` |
| `doc_id` | VARCHAR(64) | 资料批量新增返回的文档 ID |
| `business_type` | VARCHAR(32) | 业务类型（存储 docTypeId 值），根据 financeFiles/businessFiles 分类从配置获取 |
| `report_date` | VARCHAR(10) | 该文档对应的财报报告日期 |

> `reportDate` 来自请求体 `SubmitFileItem.reportDate`（前端输入），`businessType` 根据 `financeFiles` / `businessFiles` 分类从配置 `api.doc-type.*` 获取对应的 docTypeId 值。

## 四、错误码一览

| 错误码 | 触发条件 | HTTP 状态码 | 步骤 |
|--------|---------|------------|------|
| `PARAM_MISSING` | `creditCode` 或 `customerNo` 为空 | 400 | 提交资料 |
| `INVALID_CREDIT_CODE` | 信用代码格式不正确 | 400 | 提交资料 |
| `NO_FILES` | 请求体中 `financeFiles` 和 `businessFiles` 均为空 | 400 | 提交资料 |
| `INVALID_FILE_FORMAT` | 文件格式不在白名单内 | 400 | 上传附件 |
| `FILE_TOO_LARGE` | 文件超过 50MB | 400 | 上传附件 |
| `ATTACH_UPLOAD_FAILED` | 附件上传接口调用失败 | 400 | 上传附件 |
| `ATTACH_NOT_FOUND` | 请求体中的 `attId` 在 `application_att` 表中不存在 | 400 | 提交资料 |
| `BATCH_ADD_FAILED` | 资料批量新增接口调用失败 | 400 | 提交资料 |
| `DOC_NOT_FOUND` | taskId 在 application_doc 表中无关联记录 | 400 | 查询提取状态 |
| `DOC_DETAIL_FAILED` | 资料详情接口调用失败 | 400 | 查询提取状态 |
| `SYSTEM_BUSY` | 未捕获的系统异常 | 500 | 全部 |

## 五、定时清理任务

每天凌晨 2:00 自动执行，由 `AttachmentCleanupTask` 处理：

| 清理内容 | 说明 |
|---------|------|
| `application_att` 孤立附件 | 删除创建时间超过 24 小时且未关联提交的记录（用户上传后未提交资料关闭页面产生的"孤儿"数据） |
| `application_doc` 未完成任务关联记录 | 查找状态为 `UNFINISHED` 的 `application_record`，删除其下所有文档记录 |

> 对应类：`techfin-service/.../task/AttachmentCleanupTask.java`，使用 Spring `@Scheduled`（cron = `0 0 2 * * ?`）
