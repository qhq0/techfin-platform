>
> **参考 API 文档**：`api.json`（Postman Collection — dib_api）

## 一、核心业务逻辑详解

### 1.1 请求接收

#### 第一步：上传财务报表

```
POST /techfin/sxd/upload-finance-files
Content-Type: multipart/form-data
```

| 表单字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `finance_file` | File | 是 | 单个财务报表文件（PDF/Word/Excel/CSV/图片） |

**成功响应：** `{ "code": 0, "msg": "财务报表上传成功", "data": "1522260339629744128" }`

> 调用外部上传接口获取 `attId`，同时将文件元信息写入 `application_att` 表（含 attId、fileName、fileSize、businessType）。**不涉及 `application_record` 表和 reportDate**。

#### 第二步：上传商业计划书

```
POST /techfin/sxd/upload-business-files
Content-Type: multipart/form-data
```

| 表单字段 | 类型 | 必填 | 说明 |
|---------|------|------|------|
| `business_file` | File | 是 | 单个商业计划书文件（PDF/Word/PPT/图片） |

**成功响应：** `{ "code": 0, "msg": "商业计划书上传承成功", "data": "1522655358249328641" }`

> 调用外部上传接口获取 `attId`，同时将文件元信息写入 `application_att` 表（businessType 固定为 "business"）。

#### 第三步：删除附件

```
DELETE /techfin/sxd/delete-attachment/1522260339629744128
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `att_id` (路径变量) | String | 是 | 待删除的附件 ID，拼接到 URL 路径中 |

**成功响应：** `{ "code": 0, "msg": "附件删除成功", "data": null }`

> 从 `application_att` 表中删除对应记录，不影响已提交到外部系统的资料。

#### 第四步：提交资料

```
POST /techfin/sxd/submit-materials
Content-Type: application/json
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `creditCode` | String | 是 | 统一社会信用代码（18位数字或大写字母） |
| `customerNo` | String | 否 | 客户编号 |
| `financeFiles` | SubmitFileItem[] | 否 | 已上传的财务报表文件引用列表 |
| `businessFiles` | SubmitFileItem[] | 否 | 已上传的商业计划书文件引用列表 |

**SubmitFileItem 结构：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `attId` | String | 是 | 上传接口返回的附件 ID |
| `reportDate` | String | 否 | 报告日期（仅财务报表需要，来自前端输入） |

> `docName` 和 `docSize` 由服务层从 `application_att` 表根据 `attId` 查询，前端不再传入。

**请求示例：**
```json
{
  "creditCode": "91440101MA5XXXXXXX",
  "customerNo": "CUST2024001",
  "financeFiles": [
    { "attId": "1522260339629744128", "reportDate": "2024-12-31" }
  ],
  "businessFiles": [
    { "attId": "1522655358249328641" }
  ]
}
```

**成功响应：** `{ "code": 0, "msg": "成功", "data": { ... } }`

**`data` 字段结构：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `taskId` | String | 任务 ID，格式 `TASK-<32位hex>`，同时也是申请记录主键 |
| `message` | String | 提示消息 |
| `submittedCount` | Integer | 成功提交数量（固定为 1） |
| `failedIds` | FailedItem[] | 失败的 ID 列表（固定为空） |

> **鉴权说明**：下游 API 所需的 `c1-token` 统一在 `application.properties` 中配置 `api.default-token`。

## 二、数据库表结构

### application_att 表（独立实体）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT (PK) | 自增主键 |
| `att_id` | VARCHAR(64) | 附件上传返回的附件 ID |
| `file_name` | VARCHAR(255) | 上传时的原始文件名 |
| `file_size` | BIGINT | 文件大小（字节） |
| `business_type` | VARCHAR(32) | 业务类型（finance / business） |

> 该表为独立实体，上传时写入附件元信息，删除时从此表移除。提交资料时根据 `attId` 查询 `fileName` 和 `fileSize` 用于外部 API 调用。

### application_record 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | VARCHAR(64) (PK) | 任务 ID，提交时生成，格式 `TASK-<32位hex>` |
| `credit_code` | VARCHAR(18) | 统一社会信用代码 |
| `customer_no` | VARCHAR(64) | 客户编号 |
| `status` | VARCHAR(32) | 任务状态枚举（PENDING_ANALYSIS → ...） |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

> 仅在用户点击"提交资料"时创建，主键为 `task_id`。无自增 id，无外键约束。

### application_doc 表（集合表，嵌入于 application_record）

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | VARCHAR(64) | 关联 `application_record.task_id` |
| `doc_id` | VARCHAR(64) | 资料批量新增返回的文档 ID |
| `business_type` | VARCHAR(32) | 业务类型（finance / business），从对应附件记录回填 |
| `report_date` | VARCHAR(10) | 该文档对应的财报报告日期，从对应附件记录回填 |

> `reportDate` 来自请求体 `SubmitFileItem.reportDate`（前端输入），`businessType` 根据 `financeFiles` / `businessFiles` 分类确定。通过 `DocInfo.attId` 匹配回请求体数据回填到 `DocEntry`。

## 四、错误码一览

| 错误码 | 触发条件 | HTTP 状态码 | 步骤 |
|--------|---------|------------|------|
| `PARAM_MISSING` | `creditCode` 或 `customerNo` 为空 | 400 | 提交资料 |
| `INVALID_CREDIT_CODE` | 信用代码格式不正确 | 400 | 提交资料 |
| `DUPLICATE_CREDIT_CODE` | 信用代码已存在 | 400 | 提交资料 |
| `NO_FILES` | 请求体中 `financeFiles` 和 `businessFiles` 均为空 | 400 | 提交资料 |
| `INVALID_FILE_FORMAT` | 文件格式不在白名单内 | 400 | 上传文件 |
| `FILE_TOO_LARGE` | 文件超过 50MB | 400 | 上传文件 |
| `ATTACH_UPLOAD_FAILED` | 附件上传接口调用失败 | 400 | 上传文件 |
| `ATTACH_NOT_FOUND` | 请求体中的 `attId` 在 `application_att` 表中不存在 | 400 | 提交资料 |
| `BATCH_ADD_FAILED` | 资料批量新增接口调用失败 | 400 | 提交资料 |
| `SYSTEM_BUSY` | 未捕获的系统异常 | 500 | 全部 |
