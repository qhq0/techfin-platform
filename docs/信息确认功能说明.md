# 客户信息查询接口说明

> **对应功能**：根据客户编号查询企业实际控制人姓名、管户权校验
>
> **数据来源**：`sxd_profile` 表（善新贷材料生成结果表）

---

## 一、接口说明

### 1.1 请求

```
GET /techfin/sxd/controller-name/{cstId}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `cstId` (路径变量) | String | 是 | 客户编号，拼接到 URL 路径中 |

**成功响应：**

```json
{
  "code": 0,
  "msg": "成功",
  "data": "王美"
}
```

> `data` 字段直接返回实控人姓名（`actCntlrNm`）字符串。

**异常响应（客户编号不存在）：**

```json
{
  "code": -1,
  "msg": "客户编号 [CST0180779984] 不存在",
  "data": null
}
```

### 1.2 处理流程

```
① 接收请求 → 校验 cstId 非空
         → 查询 sxd_profile 表（以 cstId 为主键）
         → 返回 actCntlrNm；查不到时抛出 CUSTOMER_NOT_FOUND
```

---

### 1.3 确认/修改实控人姓名

用户在页面确认或修改实控人姓名后，回填到 `sxd_record` 表。

```
PUT /techfin/sxd/application-record/controller-name
Content-Type: application/json
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `taskId` | String | 是 | 任务 ID（submit-materials 返回的 taskId） |
| `actCntlrNm` | String | 是 | 实际控制人姓名（可修改） |

**请求示例：**

```json
{
  "taskId": "TASK-abc123...",
  "actCntlrNm": "王美"
}
```

**成功响应：**

```json
{
  "code": 0,
  "msg": "实际控制人确认成功",
  "data": null
}
```

> 根据 `taskId` 找到 `sxd_record` 记录，更新 `actCntlrNm` 和 `updatedAt` 字段。

---

### 1.4 管户权校验

校验当前登录用户是否拥有该任务对应客户的管户权。

```
POST /techfin/sxd/cust-ownership
Content-Type: application/json
```

**请求体：**

```json
{
  "taskId": "TASK-abc123...",
  "cstId": "CST0180779984"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `taskId` | String | 是 | 任务 ID，格式 `TASK-<32位hex>` |
| `cstId` | String | 是 | 客户编号 |

> 请求头需携带 `Authorization: Bearer <encrypted-token>`，后端从中解密得到 8 位用户编号。

**成功响应：**

```json
{
  "code": 0,
  "msg": "成功",
  "data": true
}
```

| 字段 | 说明 |
|------|------|
| `data` | 是否拥有管户权（判断逻辑待定，暂时恒为 true） |

**处理流程：**

```
① 接收请求 → TokenInterceptor 解密 Authorization 头获取 8 位用户编号（userId）
         → 校验 taskId、cstId 非空
         → 查询 sxd_profile 表获取 cst_mngacc_cstmgr_id、cst_mngacc_inst_supr_insid
         → 结合 userId 与管户信息判断（逻辑待定）
         → 返回 true/false
```

---

## 二、数据来源

### 数据表

| 表名 | 说明 |
|------|------|
| `sxd_profile` | 善新贷材料生成结果表，由上游 ETL 每日同步 |

### 涉及字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `cst_id` | VARCHAR(200) | 客户编号（查询条件） |
| `cst_nm` | VARCHAR(500) | 客户名称 |
| `act_cntlr_nm` | VARCHAR(200) | 实控人姓名（返回值） |
| `cst_mngacc_cstmgr_id` | VARCHAR(200) | 管户客户经理编号 |
| `cst_mngacc_inst_supr_insid` | VARCHAR(200) | 管户支行编号 |
| `data_bsn_dt` | DATE | 数据业务日期 |

> `cst_id` 为 `sxd_profile` 表的主键，每条客户编号对应唯一一条记录。

## 四、错误码一览

| 错误码 | 触发条件 | HTTP 状态码 |
|--------|---------|------------|
| `PARAM_MISSING` | `cstId` 或 `taskId` 或 `actCntlrNm` 为空 | 400 |
| `TASK_NOT_FOUND` | 任务 ID 对应的 `sxd_record` 不存在 | 400 |
| `CUSTOMER_NO_MISSING` | 任务对应的 `sxd_record` 中未找到客户编号 | 400 |
