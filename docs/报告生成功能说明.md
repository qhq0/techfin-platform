# 报告生成功能说明

> **功能入口**：要素提取 completed=true 后，前端下载 Word 报告文档
>
> **前置条件**：已完成要素提取（`GET /sxd/extract-status/{task_id}` 返回 `completed: true`）
>
> **关联文档**：[`要素提取功能说明.md`](要素提取功能说明.md)

---

## 一、核心业务逻辑

### 1.1 背景

要素提取完成后，系统已从上传的财务报表中提取了结构化的财务数据，同时商业计划书的提取文本已缓存至 `sxd_extract_data` 表。报告生成功能从这些提取数据中聚合财务报表各期间的资产负债表关键科目和利润表关键科目，同时查询**客户基本信息**（`sxd_profile` 表）、**实控人姓名和管户权**（`sxd_record` 表）和**商业计划书提取文本**（`sxd_extract_data` 表），通过 **Apache POI** 组件向 Word 模板文档的占位符位置插入数据，生成 `.docx` 格式的报告文件下载。

> **前置条件**：生成报告前需先调用 `POST /sxd/cust-ownership` 校验管户权，该接口会将结果写入 `sxd_record.has_ownership`。若 `has_ownership=0`，报告中的敏感字段将被清空。

### 1.2 生成报告接口

```
POST /techfin/sxd/report
Content-Type: application/json
```

**请求体：**

```json
{
  "taskId": "TASK-abc123...",
  "cstId": "CST0180779984"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `taskId` | String | 是 | 提交资料返回的任务 ID，格式 `TASK-<32位hex>` |
| `cstId` | String | 是 | 客户编号 |

**成功响应：** 返回 `.docx` 文件下载（`Content-Type: application/vnd.openxmlformats-officedocument.wordprocessingml.document`），文件名 `企业信息报告.docx`。

#### 表格生成规则

| 规则 | 说明 |
|------|------|
| 列标题 | 根据每个报表文档的 `report_date` 自动生成：12月31日 → `YYYY年`，其他日期 → `YYYY年M月` |
| 金额格式 | 原始数据单位为**元**，表中统一转为**万元**（÷10000），千分位保留2位小数 |
| 增长率列 | 所有增长率列统一放在表格最后，对每对相邻年末（12-31）列计算同比。计算同比：`(当期-上期)/\|上期\|×100%`，上期为0时显示 `-` |
| 资产负债率 | 资产负债表最后一行计算值：`负债合计 / 资产总计 × 100%`，单独计算增长率 |
| 毛利润率 | 利润表计算行：`(营业收入-营业成本) / 营业收入 × 100%` |
| 净利润率 | 利润表计算行：`净利润 / 营业收入 × 100%` |
| 研发费用占比 | 利润表计算行：`研发费用 / 营业收入 × 100%` |

### 1.4 判断逻辑

#### 确定使用的资产负债表类型

对每个财务报表文档，通过以下步骤确定使用哪个表查询数据：

```
┌─ 查询 docTableExtractState/{docId}
│
├─ 是否有报表附注信息（dib_intervening_y_auditreport_jh）?
│   ├─ 是 → 查询"财务报表口径"（item_value）
│   │   ├─ "1"(单一) → 使用母公司资产负债表（dib_fin_balance_parent）
│   │   └─ "2"(合并) → 使用合并资产负债表（dib_fin_balance）
│   │
│   └─ 否 → 直接比较资产负债表提取状态
│       ├─ 仅合并表为Y → 使用合并表（dib_fin_balance）
│       ├─ 仅母公司表为Y → 使用母公司表（dib_fin_balance_parent）
│       └─ 均为Y或均非Y → 默认使用合并表（dib_fin_balance）
│
└─ → 调用 queryData（{docId, tableName}）查询科目数据
```

#### 数据聚合规则

1. 每个文档对应一个 `report_date`，查询到的 `current_amount` 即为该日期的科目值
2. 多个文档（不同 `report_date`）的数据按科目名称 `item` 合并，形成完整的多日期表格
3. 表格列按 `report_date` 的先后顺序排列
4. 未匹配到数据的科目显示 `-`
5. 查询失败或返回空数组时仍输出对应日期列，该列所有项目显示 `-`

### 1.5 利润表关键科目

#### 利润表科目匹配规则

利润表科目来源与资产负债表相同的外部 `queryData` 接口，按 `item_standard` 精确匹配确定科目名称：

| 显示名称 | 匹配的 `item_standard` 值 |
|---------|------------------------|
| 营业收入 | `营业总收入` |
| 营业成本 | `营业成本` |
| 管理费用 | `管理费用` |
| 销售费用 | `销售费用` |
| 财务费用 | `财务费用` |
| 研发费用 | `研发费用` |
| 营业利润 | `营业利润` |
| 利润总额 | `利润总额` |
| 净利润 | `净利润` |

> 匹配方式为精确匹配：数据记录的 `item_standard` 字段必须与上表所列值完全一致。营业收入在 API 中标准名称为"营业总收入"，而非"营业收入"。

#### 确定使用的利润表类型

对每个财务报表文档，通过以下步骤确定使用哪个利润表查询数据：

```
┌─ 查询 docTableExtractState/{docId}
│
├─ 是否有报表附注信息（dib_intervening_y_auditreport_jh）?
│   ├─ 是 → 查询"财务报表口径"（item_value）
│   │   ├─ "1"(单一) → 使用母公司利润表（dib_fin_profit_statement_parent）
│   │   └─ "2"(合并) → 使用合并利润表（dib_fin_profit_statement）
│   │
│   └─ 否 → 直接比较利润表提取状态
│       ├─ 仅合并表为Y → 使用合并表（dib_fin_profit_statement）
│       ├─ 仅母公司表为Y → 使用母公司表（dib_fin_profit_statement_parent）
│       └─ 均为Y或均非Y → 默认使用合并表（dib_fin_profit_statement）
│
└─ → 调用 queryData（{docId, tableName}）查询科目数据
```

> 财务报表口径的判断逻辑与资产负债表完全一致（共享 `dib_intervening_y_auditreport_jh` 表口径），同一个文档的资产负债表和利润表会选择同一口径（单一→母公司表，合并→合并表）。

## 二、处理流程

```
┌─ 前端点击"生成报告"
│
├─ 后端处理流程：
│   ① 查 sxd_record → 获取实控人姓名和管户权（has_ownership）
│   ② 查 sxd_profile → 获取客户基本信息
│   ③ 根据 taskId 查 sxd_doc → 获取财务报表 docId 列表
│   ④ 对每个 docId：
│      ├─ 资产负债表：docTableExtractState → 确定合并/母公司表 → queryData
│      └─ 利润表：docTableExtractState → 确定合并/母公司表 → queryData
│   ⑤ 按科目和日期聚合数据
│   ⑥ 从 sxd_extract_data 缓存表读取商业计划书提取文本
│   ⑦ 加载 Word 模板文件 → 替换 {{占位符}}
│      ├─ has_ownership=1  → 所有字段正常填充
│      └─ has_ownership=0 → 22 个敏感字段替换为空
│   ⑧ 删除 sxd_doc 中该 task_id 的文档记录
│   ⑨ 删除 sxd_extract_data 中该 task_id 的缓存记录
│   ⑩ 更新 sxd_record.status = 1
│   ⑪ 返回 .docx 文件字节流
│
└─ 前端下载"企业信息报告.docx"
```

### 模板替换机制

报告基于 Word 模板（`.docx`）生成，模板中的占位符格式为 `{{占位符名称}}`：

- **客户信息字段**：`{{cst_nm}}`、`{{credit_code}}` 等 34 个占位符 → 替换为字段值
- **商业计划书提取文本**：`{{dib_manage_company_profile}}` 等 9 个占位符 → 替换为缓存表中的提取文本
- **资产负债表表格**：`{{资产负债表关键科目}}` → 在该位置插入 XWPFTable
- **利润表表格**：`{{利润表关键科目}}` → 在该位置插入 XWPFTable

代码通过 `XmlCursor` 在占位符段落位置插入表格，然后移除占位符段落。用户可直接替换模板文件，只要保留上述占位符即可。

| 项目 | 说明 |
|------|------|

### 提取数据查询（报表附注）`POST /api/extract/open/doc/queryData`

`tableName` 为 `dib_intervening_y_auditreport_jh`，用于查询"财务报表口径"。

Response data 数组元素：

```json
{
  "item": "财务报表口径",
  "item_value": "2",
  "report_date": "2025-06-30"
}
```

| 字段 | 说明 |
|------|------|
| `item` | 项目名称，`财务报表口径` 或 `财务报表截止日期` |
| `item_value` | `1`=单一（母公司表），`2`=合并（合并表） |

| 接口 | 解析方式 | DTO |

| `queryData`（利润表） | `getDataAsList(BalanceSheetRecord.class)` | `BalanceSheetRecord` (item, itemStandard, currentAmount, reportDate) |
| `queryData`（报表附注） | `getDataAsList(AuditReportItem.class)` | `AuditReportItem` (item, itemValue) |

## 五、数据模型

### sxd_record 表（申请记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | VARCHAR(64) (PK) | 任务 ID |
| `credit_code` | VARCHAR(30) | 统一社会信用代码 |
| `cst_id` | VARCHAR(64) | 客户编号 |
| `status` | VARCHAR(10) | 任务状态 |

### sxd_doc 表（文档明细表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `doc_id` | VARCHAR(64) (PK) | 资料批量新增返回的文档 ID |
| `task_id` | VARCHAR(64) | 关联 `sxd_record.task_id` |
| `business_type` | VARCHAR(32) | 业务类型（docTypeId），finance=1273757631042949120 |
| `report_date` | VARCHAR(10) | 财报报告日期，作为表格列标题来源 |

### sxd_extract_data 表（提取数据缓存表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT (PK, AUTO_INCREMENT) | 自增主键 |
| `task_id` | VARCHAR(64) | 关联 `sxd_record.task_id` |
| `doc_id` | VARCHAR(64) | 关联 `sxd_doc.doc_id` |
| `table_name` | VARCHAR(128) | 提取表名，如 `dib_manage_company_profile` |
| `text` | TEXT | 提取文本内容 |

> 报告生成时从该表读取商业计划书的 9 个 tableName 对应的提取文本，取最新记录。若缓存表中无数据（如未调用提取数据查询接口），占位符替换为空字符串。

## 六、配置项

```properties
# 外部表格提取状态接口地址（不含 doc_id，调用时拼接）
api.doc-table-extract-state-url=${api.base-url}/api/extract/open/doc/docTableExtractState

# Word 报告模板文件路径（支持 classpath: 前缀）
```

| 属性 | 说明 |
|------|------|
| `api.doc-table-extract-state-url` | 表格提取状态接口基地址，调用时在末尾拼接 `/{doc_id}` |
| `api.report-template-path` | 报告模板路径。支持 `classpath:`（默认）和 `file:` 外部路径 |

> 其他共享配置（`api.doc-query-data-url`、`api.default-token` 等）参见[要素提取功能说明](要素提取功能说明.md#六配置项)。

## 七、错误码

| 错误码 | 触发条件 | HTTP 状态码 |
|--------|---------|------------|
| `TASK_NOT_FOUND` | `taskId` 在 `sxd_record` 表中不存在 | 400 |
| `DOC_NOT_FOUND` | `taskId` 在 `sxd_doc` 表中无关联的财务报表记录 | 400 |
| `TABLE_STATE_FAILED` | 外部表格提取状态接口调用失败（网络异常 / 非成功响应） | 400 |
| `DOC_QUERY_FAILED` | 外部提取数据查询接口（资产负债表或利润表）调用失败 | 400 |
| `NO_BALANCE_SHEET_DATA` | 所有文档均未查询到有效的资产负债表数据 | 400 |
| `TEMPLATE_NOT_FOUND` | 模板文件不存在（检查 `api.report-template-path` 路径） | 400 |
| `DOC_CREATE_FAILED` | Word 文档生成异常（POI 写入失败） | 400 |

## 八、与要素提取功能的关系

```
要素提取功能                     报告生成功能
─────────────────               ────────────────
GET /extract-status/{task_id}
  → completed=true
GET /extract-data/business/{task_id}
  → 提取文本缓存至 sxd_extract_data
                                  POST /sxd/report
                                    → 查客户信息 → 查财务报表
                                    → 查 sxd_extract_data 缓存
                                    → docTableExtractState → 确定表类型
                                    → queryData(资产负债表) → 获取科目
                                    → queryData(利润表) → 获取科目
                                    → 加载模板 → 替换占位符 → 插入表格
                                    → 返回 .docx 下载

GET /export-data/business/{task_id}
GET /export-data/finance/{task_id}
（下载原始 xlsx）
```

> 报告生成功能依赖于要素提取完成（`completed=true`），复用同一套 `sxd_doc` 和 `queryData` 外部 API。不同之处在于：导出功能下载原始 xlsx 文件，报告功能查询结构化数据并聚合成 Word 文档。
