# 报告生成功能说明

> **功能入口**：要素提取 completed=true 后，前端点击"生成报告"按钮
>
> **前置条件**：已完成要素提取（`GET /sxd/extract-status/{task_id}` 返回 `completed: true`）
>
> **关联文档**：[`要素提取功能说明.md`](要素提取功能说明.md)

---

## 一、核心业务逻辑

### 1.1 背景

要素提取完成后，系统已从上传的财务报表中提取了结构化的财务数据。报告生成功能从这些提取数据中聚合财务报表各期间的资产负债表关键科目和利润表关键科目，生成 markdown 格式的表格报告供前端展示。

### 1.2 生成报告接口

```
GET /techfin/sxd/report/{task_id}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_id`（路径变量） | String | 是 | 提交资料返回的任务 ID，格式 `TASK-<32位hex>` |

**成功响应：**

```json
{
  "code": 0,
  "msg": "成功",
  "data": "## 资产负债表关键科目\n\n| 项目 | 2025年9月 | 2024年 | 2023年 | 2022年 | 2024年增长率 |\n| --- | --- | --- | --- | --- | --- |\n| ..."
```

#### 表格生成规则

| 规则 | 说明 |
|------|------|
| 列标题 | 根据每个报表文档的 `report_date` 自动生成：12月31日 → `YYYY年`，其他日期 → `YYYY年M月` |
| 金额格式 | 原始数据单位为**元**，表中统一转为**万元**（÷10000），千分位保留2位小数 |
| 增长率列 | 从所有列中筛选年末（12-31）列，取倒数第二个年末列与最后一个年末列计算同比：`(当期-上期)/\|上期\|×100%`，上期为0时显示 `-`，列标题以最后一个年末列的年份命名（如 `2024年增长率`） |
| 资产负债率 | 资产负债表最后一行计算值：`负债合计/资产总计×100%`，单独计算增长率 |
| 毛利润率 | 利润表计算行：`(营业收入-营业成本)/营业收入×100%` |
| 净利润率 | 利润表计算行：`净利润/营业收入×100%` |
| 研发费用占比 | 利润表计算行：`研发费用/营业收入×100%` |

### 1.3 判断逻辑

#### 确定使用的资产负债表类型

对每个财务报表文档，通过以下步骤确定使用哪个表查询数据：

```
┌─ 查询 docTableExtractState/{docId}
│
├─ 是否有报表附注信息（dib_intervening_y_auditreport_jh）?
│   ├─ 是 → 查询"财务报表口径"（item_value）
│   │   ├─ "1"(单一) → 使用母公司资产负债表（dib_fin_balance_parent）
│   │   └─ "2"(合并) → 使用合并资产负债表（dib_fin_balance）
│   │
│   └─ 否 → 直接比较资产负债表提取状态
│       ├─ 仅合并表为Y → 使用合并表（dib_fin_balance）
│       ├─ 仅母公司表为Y → 使用母公司表（dib_fin_balance_parent）
│       └─ 均为Y或均非Y → 默认使用合并表（dib_fin_balance）
│
└─ → 调用 queryData（{docId, tableName}）查询科目数据
```

#### 数据聚合规则

1. 每个文档对应一个 `report_date`，查询到的 `current_amount` 即为该日期的科目值
2. 多个文档（不同 `report_date`）的数据按科目名称 `item` 合并，形成完整的多日期表格
3. 表格列按 `report_date` 的先后顺序排列
4. 未匹配到数据的科目显示 `-`

### 1.4 利润表关键科目

#### 利润表科目匹配规则

利润表科目来源与资产负债表相同的外部 `queryData` 接口，按 `item_standard` 精确匹配确定科目名称：

| 显示名称 | 匹配的 `item_standard` 值 |
|---------|------------------------|
| 营业收入 | `营业总收入` |
| 营业成本 | `营业成本` |
| 管理费用 | `管理费用` |
| 销售费用 | `销售费用` |
| 财务费用 | `财务费用` |
| 研发费用 | `研发费用` |
| 营业利润 | `营业利润` |
| 利润总额 | `利润总额` |
| 净利润 | `净利润` |

> 匹配方式为精确匹配：数据记录的 `item_standard` 字段必须与上表所列值完全一致。营业收入在 API 中标准名称为"营业总收入"，而非"营业收入"。

#### 确定使用的利润表类型

对每个财务报表文档，通过以下步骤确定使用哪个利润表查询数据：

```
┌─ 查询 docTableExtractState/{docId}
│
├─ 是否有报表附注信息（dib_intervening_y_auditreport_jh）?
│   ├─ 是 → 查询"财务报表口径"（item_value）
│   │   ├─ "1"(单一) → 使用母公司利润表（dib_fin_profit_statement_parent）
│   │   └─ "2"(合并) → 使用合并利润表（dib_fin_profit_statement）
│   │
│   └─ 否 → 直接比较利润表提取状态
│       ├─ 仅合并表为Y → 使用合并表（dib_fin_profit_statement）
│       ├─ 仅母公司表为Y → 使用母公司表（dib_fin_profit_statement_parent）
│       └─ 均为Y或均非Y → 默认使用合并表（dib_fin_profit_statement）
│
└─ → 调用 queryData（{docId, tableName}）查询科目数据
```

> 财务报表口径的判断逻辑与资产负债表完全一致（共享 `dib_intervening_y_auditreport_jh` 表口径），同一个文档的资产负债表和利润表会选择同一口径（单一→母公司表，合并→合并表）。

## 二、处理流程

```
┌─ 前端点击"生成报告"
│
├─ 后端处理流程：
│   ① 根据 taskId 查 application_doc 获取财务报表的 docId 列表
│   ② 对每个 docId：
│      ├─ 资产负债表：调用 docTableExtractState → 确定合并/母公司表 → queryData
│      └─ 利润表：调用 docTableExtractState → 确定合并/母公司表 → queryData
│   ③ 按科目和日期聚合，分别生成资产负债表和利润表数据
│   ④ 筛选年末（12-31）列，取最后两个年末列计算各科目同比增长率
│   ⑤ 生成 markdown 文本（资产负债表表格 + 利润表表格）返回
│
└─ 前端展示：渲染 markdown 中的两个表格
```

### 提取数据查询（资产负债表）`POST /api/extract/open/doc/queryData`

同[要素提取功能说明 — 提取数据查询](要素提取功能说明.md#提取数据查询-post-apiextractopendocquerydata)，`tableName` 为 `dib_fin_balance` 或 `dib_fin_balance_parent`。

Response data 数组元素：

```json
{
  "item_code": "B1100101BB",
  "item": "货币资金",
  "current_amount": 12345678901.23,
  "last_amount": 98765432109.87,
  "report_date": "2025-06-30",
  "unit": "元"
}
```

### 提取数据查询（利润表）`POST /api/extract/open/doc/queryData`

`tableName` 为 `dib_fin_profit_statement` 或 `dib_fin_profit_statement_parent`。

Response data 数组元素结构与资产负债表相同：

```json
{
  "item_code": "P1000001PT",
  "item": "一、营业总收入",
  "item_standard": "营业总收入",
  "current_amount": 91093762553.97,
  "last_amount": 83451164646.53,
  "report_date": "2025-06-30",
  "unit": "元"
}
```

> 利润表与资产负债表共享同一个 `queryData` 接口，仅 `tableName` 参数不同。响应结构完全一致，后端复用 `BalanceSheetRecord` DTO 解析，并通过 `item_standard` 字段进行科目匹配。

### 提取数据查询（报表附注）`POST /api/extract/open/doc/queryData`

`tableName` 为 `dib_intervening_y_auditreport_jh`，用于查询"财务报表口径"。

Response data 数组元素：

```json
{
  "item": "财务报表口径",
  "item_value": "2",
  "report_date": "2025-06-30"
}
```

| 字段 | 说明 |
|------|------|
| `item` | 项目名称，`财务报表口径` 或 `财务报表截止日期` |
| `item_value` | `1`=单一（母公司表），`2`=合并（合并表） |

### 响应解析对照

| 接口 | 解析方式 | DTO |
|------|---------|-----|
| `docTableExtractState` | `getDataAsList(DocTableStateRecord.class)` | `DocTableStateRecord` (tableName, extractState) |
| `queryData`（资产负债表） | `getDataAsList(BalanceSheetRecord.class)` | `BalanceSheetRecord` (item, itemStandard, currentAmount, reportDate) |
| `queryData`（利润表） | `getDataAsList(BalanceSheetRecord.class)` | `BalanceSheetRecord` (item, itemStandard, currentAmount, reportDate) |
| `queryData`（报表附注） | `getDataAsList(AuditReportItem.class)` | `AuditReportItem` (item, itemValue) |

## 四、数据模型

### application_doc 表（文档明细表）

| 字段 | 类型 | 说明 |
|------|------|------|
| `doc_id` | VARCHAR(64) (PK) | 资料批量新增返回的文档 ID |
| `task_id` | VARCHAR(64) | 关联 `application_record.task_id` |
| `business_type` | VARCHAR(32) | 业务类型（docTypeId），finance=1273757631042949120 |
| `report_date` | VARCHAR(10) | 财报报告日期，作为表格列标题来源 |

### 关键 DTO（本功能新增）

| 类 | 位置 | 说明 |
|----|------|------|
| `DocTableStateRecord` | `model/.../dto/external/` | 表格提取状态响应 data 元素 |
| `BalanceSheetRecord` | `model/.../dto/external/` | 资产负债表/利润表 queryData 响应 data 元素（item, itemStandard, currentAmount） |
| `AuditReportItem` | `model/.../dto/external/` | 审计报告附注 queryData 响应 data 元素（item+itemValue） |

## 六、错误码

| 错误码 | 触发条件 | HTTP 状态码 |
|--------|---------|------------|
| `DOC_NOT_FOUND` | `taskId` 在 `application_doc` 表中无关联的财务报表记录 | 400 |
| `TABLE_STATE_FAILED` | 外部表格提取状态接口调用失败（网络异常 / 非成功响应） | 400 |
| `DOC_QUERY_FAILED` | 外部提取数据查询接口（资产负债表或利润表）调用失败 | 400 |
| `NO_BALANCE_SHEET_DATA` | 所有文档均未查询到有效的资产负债表数据 | 400 |

## 七、与要素提取功能的关系

```
要素提取功能                     报告生成功能
─────────────────               ────────────────
GET /extract-status/{task_id}
  → completed=true
                                  GET /sxd/report/{task_id}
                                    → docTableExtractState → 确定表类型
                                    → queryData(资产负债表) → 获取科目
                                    → queryData(利润表) → 获取科目
                                    → 聚合 → 两个 markdown 表格

GET /export-data/business/{task_id}
GET /export-data/finance/{task_id}
（下载原始 xlsx）
```

> 报告生成功能依赖于要素提取完成（`completed=true`），复用同一套 `application_doc` 和 `queryData` 外部 API。不同之处在于：导出功能下载原始 xlsx 文件，报告功能查询结构化数据并聚合成 markdown 表格。
